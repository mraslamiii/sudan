# لیست کامل امکانات سیستم هوشمند

این فایل شامل تمام امکانات و قابلیت‌های سیستم هوشمند است که باید در پروژه پیاده‌سازی شوند.

---

## ۱. کنترل روشنایی‌ها

- امکان تعریف آیکون انتخابی و فراخوانی از حافظه‌ی تبلت
- قابلیت تفکیک طبقه، مکان و سرخط‌ها
- قابلیت تعریف سناریوها برای روشنایی هر مکان و طبقه

**پروتکل:** `SocketConstants.headLineLight = 'U'`

---

## ۲. کنترل دما و تهویه

- کنترل دما، سرمایش و گرمایش هر طبقه از طریق تبلت

**پروتکل:** `SocketConstants.headLineTemperature = 'W'`

---

## ۳. احضار آسانسور

- کاربر می‌تواند از طریق تبلت، آسانسور را فراخوانی کند
- امکان ارسال دستور احضار برای هر طبقه

---

## ۴. ارتباط با آیفون و دوربین‌ها

- قابلیت ارتباط با دوربین آیفون (آنالوگ یا IP) دم درب حیاط
- امکان باز کردن درب از طریق دوربین
- ارتباط با دوربین اضافه‌شده دم واحد
- باز کردن درب از موقعیت دوربین واحد

**پروتکل:** `SocketConstants.headLineCameras = 'X2'`

---

## ۵. ارتباط داخلی و مدیریت تماس‌ها

- امکان ارتباط بین آپارتمان‌ها با یکدیگر
- پنل مدیریت با قابلیت برقراری ارتباط با تمامی واحدها
- ارتباط با نگهبانی از طریق تماس
- امکان گذاشتن پیام

---

## ۶. بخش امنیت (دزدگیر)

- قابلیت فعال و غیرفعال کردن دزدگیر
- انجام تنظیمات دزدگیر
- بارگذاری شماره تماس‌ها و پیام‌ها
- تغییر پسورد
- اجرای تایمر زمان و فعال شدن دزدگیر از طریق تبلت

**پروتکل:** `SocketConstants.headLineBurglarAlarm = 'X1'`
- `SocketConstants.burglarAlarmIsOn = 'on'`
- `SocketConstants.burglarAlarmIsOff = 'of'`
- `SocketConstants.requestBurglarAlarms = '@X1Z'`

---

## ۷. موزیک پلیر

- قابلیت قطع و وصل کردن ده بلندگوی مجزا
- امکان زمان‌دهی برای فعال‌سازی موسیقی انتخابی
- تنظیم زمان (ساعت، روز و ماه) برای پخش موسیقی

---

## ۸. اعلان‌ها و پیام‌ها

- مدیریت ارسال و دریافت پیامک
- مدیریت پیام‌های ارسال‌شده به تبلت
- قابلیت گذاشتن اعلان در زمان خاص (ساعت، روز و ماه) برای ساکنین منزل

---

## ۹. سناریوها

- امکان تعریف سناریوهای مختلف
- کنترل همزمان چند عملکرد (روشنایی، دما، موسیقی و غیره)
- اجرای سناریو در مکان‌ها و طبقات مختلف

**پروتکل:**
- `SocketConstants.commandScenarioGeneral = '!&'` - سناریو عمومی
- `SocketConstants.commandScenarioFloor = '!^'` - سناریو طبقه
- `SocketConstants.commandScenarioPlace = '!~'` - سناریو مکان
- `SocketConstants.headLineScenarios = 'X'`

---

## ۱۰. دستیار صوتی

- دستیار صوتی جهت اجرای فرامین صوتی
- از موبایل به تبلت
- ورودی AUX تبلت

---

## ۱۱. سخت‌افزار کنترل

- تبلت‌های مورد استفاده بین ۷ تا ۱۰ اینچ
- کاربر می‌تواند مکان، طبقه، سرخط‌ها و آیکون‌ها را به‌صورت دلخواه پیکربندی کند
- امکان تعریف آیکون‌های سفارشی

---

## ۱۲. تنظیمات و تبادل داده

- امکان تعریف و مدیریت راه‌نماها
- تنظیمات عمومی و اختصاصی سیستم
- تبادل دوطرفه‌ی داده بین تبلت و میکروکنترلر
  - از تبلت به میکرو
  - از میکرو به تبلت
- ارتباط بین تبلت و میکروکنترلر از طریق کابل OTG

**پروتکل:**
- `SocketConstants.requestIp = '@M_IP'` - درخواست IP
- `SocketConstants.requestQueryFloorsCount = '@M_F_C'` - درخواست تعداد طبقات
- `SocketConstants.requestAFloor = '@M_F_'` - درخواست اطلاعات یک طبقه

---

## ۱۳. اجرای خودکار نرم‌افزار

- نرم‌افزار سیستم به‌گونه‌ای طراحی شده که بلافاصله پس از روشن شدن تبلت، به‌صورت خودکار (Boot) اجرا می‌شود

---

## ۱۴. فرمان شارژ و دشارژ تبلت

- قابلیت کنترل فرمان شارژ و دشارژ تبلت
- حفظ سلامت باتری
- پایداری عملکرد دستگاه در بلندمدت

---

## پروتکل‌های ارتباطی

### پیشوندهای دستورات
- `&` - Command
- `@` - Request
- `%` - Start IP Config
- `#` - Start Data
- `*` - Start Place
- `/` - New Object

### کدهای Headline
- `U` - Light (روشنایی)
- `V` - Curtain (پرده)
- `W` - Temperature (دما)
- `X` - Scenarios (سناریوها)
- `X1` - Burglar Alarm (دزدگیر)
- `X2` - Cameras (دوربین‌ها)

### تنظیمات پیش‌فرض
- IP پیش‌فرض: `192.168.4.1`
- Port پیش‌فرض: `6269`

## نکات مهم پیاده‌سازی

1. تمام ارتباطات از طریق TCP Socket انجام می‌شود
2. استفاده از معماری MVVM برای تمام بخش‌ها
3. مدیریت state با ViewModel ها
4. استفاده از Repository Pattern برای دسترسی به داده
5. استفاده از Use Cases برای منطق کسب‌وکار
6. پشتیبانی از تم روشن و تاریک
7. مدیریت خطا و نمایش پیام‌های مناسب
8. پشتیبانی از چند زبان (در صورت نیاز)
